# 🎯 روليت السحب العشوائي

صفحة مستقلة لإجراء سحوبات عشوائية عادلة بين مجموعة من المشاركين (اسم + رقم وظيفي).  
تُفتح مباشرةً بالنقر المزدوج على `index.html` — لا تحتاج إلى خادم.

---

## 🚀 التشغيل

1. افتح الملف `index.html` مباشرة في متصفح حديث (Chrome, Edge, Firefox, Safari).
2. لا توجد أي إعدادات أو تثبيت — الصفحة تعمل بشكل مستقل.
3. يحتفظ المتصفح بالمشاركين والسجل تلقائياً عبر `localStorage`.

> **ملاحظة:** إذا فتحتها عبر خادم محلي (VSCode Live Server / Python `http.server`)، ستعمل بشكل أفضل من ناحية بعض حواف المتصفح، لكنها لا تشترط ذلك.

## ✨ المزايا المنفذة

### 1) إدخال المشاركين
- حقل **الاسم** وحقل **الرقم الوظيفي** — كلاهما إجباري.
- رسائل خطأ واضحة بجانب كل حقل.
- **منع تكرار الرقم الوظيفي** تماماً.
- تنظيف المدخلات تلقائياً (إزالة الفراغات الزائدة).
- تعديل أي مشارك بالنقر على أيقونة التعديل.
- حذف أي مشارك قبل السحب.
- تصفير القائمة بالكامل بتأكيد.

### 2) عجلة الروليت
- مرسومة على **HTML Canvas** بدقة DPR (عالية الوضوح على شاشات Retina).
- تُولّد الألوان تلقائياً من لوحة متناسقة مع الثيم الداكن.
- اسم المشارك + رقمه الوظيفي يظهران داخل كل قطاع.
- حجم الخط يتكيف تلقائياً مع عدد المشاركين.
- حلقة ذهبية خارجية مع نقاط زخرفية تدور مع العجلة.
- مؤشر (Pointer) ذهبي في أعلى العجلة.

### 3) آلية السحب العشوائي 🔐
**هذا هو القلب العادل للنظام:**

```js
function secureRandomIndex(max) {
  const buf = new Uint32Array(1);
  const MAX_U32 = 0xFFFFFFFF;
  const limit = MAX_U32 - (MAX_U32 % max) - 1;
  let r;
  do {
    crypto.getRandomValues(buf);  // مصدر عشوائية تشفيري
    r = buf[0];
  } while (r > limit);             // رفض العينات لتجنب انحياز القسمة
  return r % max;
}
```

- نستخدم `crypto.getRandomValues` — واجهة عشوائية **تشفيرية آمنة** من المتصفح، وليس `Math.random`.
- نستخدم **Rejection Sampling** لتجنب انحياز عملية modulo (Modulo Bias) — وهذا يضمن فرصة متساوية **تماماً** لكل مشارك حتى عندما لا يقسم عدد المشاركين `2^32` بالتساوي.
- **الفائز يُختار أولاً** (عبر العشوائية الحقيقية)، ثم تُحسب الزاوية النهائية للعجلة ليستقر المؤشر عليه — لا يحدد الفوز توقيت الضغط على زر الإيقاف.
- يُضاف انحراف بصري صغير (±30% من القطاع) ليبدو التوقف طبيعياً غير مصطنع.
- 7–9 دورات كاملة إضافية لإعطاء إحساس الإثارة.

> **الخلاصة:** الفائز يُعيَّن بطريقة عشوائية حقيقية قبل أن تبدأ العجلة الدوران — لا يوجد أي تحكم يدوي أو تمييز.

### 4) خيار استبعاد الفائز
- عند تفعيل الخيار وإعادة السحب، يُستبعد الفائز السابق تلقائياً من القائمة المؤهلة — مفيد للسحوبات المتعددة (مراكز أولى / ثانية / إلخ).
- يمكن إيقاف الخيار فيعود الجميع مؤهلين.

### 5) بطاقة الفائز
- تظهر بعد توقف العجلة مع تأثير مرئي (تكبير مرن).
- تعرض: الاسم، الرقم الوظيفي، وقت السحب بالثانية.
- تأثير كونفيتي Confetti ملوّن.
- زر **سحب مرة أخرى** لإجراء سحب جديد مباشرة.

### 6) سجل السحوبات
- كل سحب يُحفظ تلقائياً في `localStorage`:
  - اسم الفائز
  - الرقم الوظيفي
  - تاريخ + وقت السحب
  - عدد المشاركين وقت السحب
- يحتفظ بآخر 100 سحب (لمنع الانتفاخ).
- يمكن فتح السجل من زر أعلى الصفحة.
- يمكن مسحه بالكامل بتأكيد.

### 7) الأمان والتحقق
- تنظيف النصوص قبل الحفظ (`trim` + إزالة الفراغات المكررة).
- Escape HTML للمحتوى عند عرضه (حماية من XSS حتى لو أُدخل نص يشبه HTML).
- منع السحب إذا كانت القائمة فارغة.
- منع الضغط المتكرر على زر السحب أثناء التنفيذ (الزر يُعطَّل + class على body).
- قفل نموذج الإدخال بصرياً أثناء الدوران.
- تأكيد إضافي قبل الحذف أو التصفير.
- مفتاح `Esc` لإغلاق الطبقات الفوقية.

### 8) التصميم والتجاوب
- RTL كامل مع خط **Cairo** (عربي).
- يعمل بسلاسة على:
  - 📱 الجوال (≤600px)
  - 💻 الكمبيوتر (≥981px)
  - 📏 الشاشات الطولية / اللوحية
- مؤشر حالة واضح: **جاهز / جاري السحب / تم اختيار الفائز**.
- دعم `prefers-reduced-motion` لتقليل الرسوم المتحركة لمن يحتاجها.

## 🔧 التخصيص السريع

### تغيير لوحة ألوان العجلة
عدّل المصفوفة في `script.js`:
```js
const SEGMENT_COLORS = ['#FFD166', '#EF476F', /* ... */];
```

### تغيير مدة الدوران
في `script.js` → دالة `performDraw`:
```js
const duration = 5500 + Math.floor(Math.random() * 800); // مللي ثانية
```

### تغيير عدد الدورات
```js
const extraSpins = 7 + Math.floor(Math.random() * 3);
```

### تغيير لون الثيم الأساسي
في `styles.css` → جذر `:root`:
```css
--gold: #FFD166;
--gold-dark: #F5B638;
```

## 🔌 ربط قاعدة البيانات (للمرحلة الثانية)

النسخة الحالية تستخدم **`localStorage`** (تخزين محلي داخل المتصفح فقط).

إذا طُلب الربط بقاعدة بيانات رسمية لاحقاً، النقاط المطلوب استبدالها في `script.js`:

| الدالة الحالية | الاستبدال المقترح |
|---------------|------------------|
| `saveParticipants()` | `POST /api/participants` |
| `loadState()` | `GET /api/participants` + `GET /api/draws` |
| `saveHistory()` جزء `history.unshift(entry)` | `POST /api/draws` |
| `clearAllParticipants()` | `DELETE /api/participants` |

كل نقاط الإدخال/الإخراج في الكود معزولة داخل قسم `Persistence` المعلّم بوضوح — الاستبدال مباشر.
