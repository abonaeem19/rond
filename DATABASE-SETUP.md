# 🗄️ إعداد قاعدة البيانات — Supabase

## ما الفائدة؟
- تسجيل المشاركين من **جهاز** والسحب من **جهاز آخر**
- البيانات تتحدث **تلقائياً** بين الأجهزة (Real-time)
- حفظ **دائم** للمشاركين وسجل السحوبات
- **مجاني** بالكامل (Supabase Free Tier)

---

## الخطوات (مرة واحدة فقط)

### 1️⃣ إنشاء حساب Supabase
1. ادخل [supabase.com](https://supabase.com) واضغط **Start your project**
2. سجّل بحساب GitHub أو بريدك الإلكتروني

### 2️⃣ إنشاء مشروع جديد
1. اضغط **New project**
2. اختر اسم (مثلاً: `roulette`)
3. اختر كلمة مرور قوية (لقاعدة البيانات)
4. اختر المنطقة الأقرب لك (مثلاً: `Middle East (Bahrain)`)
5. اضغط **Create new project** وانتظر دقيقة

### 3️⃣ إنشاء الجداول
1. من القائمة الجانبية اختر **SQL Editor**
2. اضغط **New query**
3. افتح ملف `supabase-setup.sql` من مجلد المشروع
4. انسخ كل المحتوى والصقه في المحرر
5. اضغط **Run** (الزر الأخضر)
6. يجب أن ترى ✅ `Success. No rows returned`

### 4️⃣ نسخ بيانات الاتصال
1. من القائمة الجانبية اختر **Settings** (الترس) → **API**
2. انسخ:
   - **Project URL** — مثلاً: `https://abcdefgh.supabase.co`
   - **anon public** key — مفتاح طويل يبدأ بـ `eyJhbGci...`

### 5️⃣ تعديل ملف db.js
افتح ملف `rond/db.js` وعدّل السطرين في الأعلى:

```javascript
const SUPABASE_URL  = 'https://abcdefgh.supabase.co';  // ← ضع رابطك هنا
const SUPABASE_KEY  = 'eyJhbGci...your-anon-key';       // ← ضع المفتاح هنا
```

### 6️⃣ جرّب!
1. افتح الصفحة — يجب أن يظهر **"متصل"** بجانب الستيبر
2. سجّل مشاركين من جهازك
3. افتح نفس الرابط من جهاز آخر (جوال مثلاً)
4. ستجد المشاركين موجودين!

## كيف يعمل النظام

```
جهاز 1 (التسجيل)          Supabase Cloud          جهاز 2 (السحب)
    │                           │                       │
    ├── إضافة مشارك ──────────▶ │ ◀── Realtime ────────▶ │ يتحدث تلقائياً
    ├── تعديل / حذف ──────────▶ │                       │
    │                           │ ◀────── تحميل ────────┤ السحب + النتيجة
    │                           │ ──────▶ حفظ ──────────▶ │
```

- **Realtime**: أي تغيير من جهاز يظهر فوراً في الجهاز الآخر (بدون تحديث)
- **Fallback**: إذا انقطع الإنترنت، يشتغل من localStorage كالعادة

## البنية

| الجدول | الأعمدة | الوظيفة |
|--------|---------|---------|
| `participants` | `id`, `name`, `emp_id`, `created_at` | قائمة المشاركين |
| `draws` | `id`, `winner_name`, `winner_emp_id`, `total_participants`, `drawn_at` | سجل السحوبات |

## ملاحظات مهمة
- المفتاح `anon public` **آمن** للاستخدام في الواجهة — لا يمنح صلاحيات خطيرة
- صلاحيات الجداول مفتوحة (RLS) لأن التطبيق عام ولا يحتاج تسجيل دخول
- Supabase Free Tier يدعم حتى 500MB بيانات و50,000 طلب شهرياً — أكثر من كافٍ
- إذا أردت تقييد الوصول لاحقاً، يمكنك تعديل سياسات RLS في Supabase Dashboard
